# PRD — 任務管理系統（Task Manager）

## 1. 專案概述

### 1.1 背景與動機

在日常工作與學習中，人們需要一個簡單、直觀的工具來記錄待辦事項並追蹤完成進度。
本系統旨在提供一個輕量級的任務管理介面，讓使用者能快速新增、管理與完成任務，無需複雜的設定或學習成本。

### 1.2 目標用戶

- 個人使用者（學生、上班族）需要追蹤每日待辦事項
- 小型團隊需要簡易任務清單共享（未來擴充）

### 1.3 核心價值主張

- **簡單易用**：無需帳號，立即開始使用
- **即時回饋**：新增、完成、刪除任務皆有立即視覺反應
- **清晰可見**：一頁顯示所有任務狀態，一目了然

## 2. 功能需求

### 2.1 主要功能清單

| # | 功能名稱 | 說明 |
|---|----------|------|
| F1 | 新增任務 | 使用者輸入任務名稱並送出，新任務出現在清單中 |
| F2 | 顯示任務清單 | 顯示所有任務，包含狀態（待完成 / 已完成） |
| F3 | 標記任務完成 | 點擊後將任務標記為已完成，並以刪除線視覺區隔 |
| F4 | 刪除任務 | 將任務從清單中永久移除 |
| F5 | 任務狀態篩選 | 可切換查看「全部 / 待完成 / 已完成」任務 |

### 2.2 使用者故事

**F1 — 新增任務**
> 作為一位使用者，我希望能輸入任務名稱並按下送出按鈕，以便將新的待辦事項加入清單。

**F2 — 顯示任務清單**
> 作為一位使用者，我希望進入頁面時就能看到所有任務的清單與狀態，以便快速掌握目前的工作進度。

**F3 — 標記任務完成**
> 作為一位使用者，我希望能點擊任務旁的勾選按鈕，以便將已完成的任務標記起來，與未完成的任務區分。

**F4 — 刪除任務**
> 作為一位使用者，我希望能對任意任務點擊刪除按鈕，以便移除不再需要的任務，保持清單整潔。

**F5 — 狀態篩選**
> 作為一位使用者，我希望能透過篩選按鈕切換查看不同狀態的任務，以便專注處理特定類型的任務。

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML + CSS（Vanilla）+ 少量 JavaScript |
| 部署環境 | 本機開發（localhost） |

### 3.2 效能考量

- 頁面首次載入時間應在 **1 秒內**（本機環境）
- 新增 / 刪除 / 標記操作應在 **500ms 內** 完成並更新頁面

### 3.3 安全考量

- 輸入欄位需防止 **XSS 注入**（透過 Jinja2 自動跳脫）
- 資料庫操作使用 **參數化查詢**，防止 SQL Injection
- 任務名稱長度限制：最多 **200 個字元**

### 3.4 使用體驗需求

- 支援中文輸入
- 響應式設計，支援桌面與平板瀏覽
- 空清單時顯示引導提示文字
